# Запуск формы создания проекта (Windows)

Инструкция для консультанта: как заполнить форму и создать проект в Jira без работы с кодом напрямую.

Команды ниже — для **PowerShell** (стандартный терминал Windows). Если открыт `cmd.exe`,
команда `copy` работает так же, `python` — так же.

## 0. Один раз: проверить, что Python установлен

```powershell
python --version
```

Если появилась ошибка «python не является внутренней или внешней командой» — установить
Python с [python.org](https://www.python.org/downloads/) и при установке обязательно
поставить галочку **«Add python.exe to PATH»**.

На Windows команда называется **`python`**, не `python3` (в отличие от macOS/Linux).

## 1. Один раз перед первым запуском

```powershell
cd e-sklep-b2b\agent
copy .env.example .env
```

Открыть `.env` в Блокноте и вписать:

```
JIRA_BASE_URL=https://<ваш-домен>.atlassian.net
JIRA_EMAIL=<ваш email в Jira>
JIRA_API_TOKEN=<токен Jira>
```

Токен создаётся в Jira: **Account Settings → Security → API tokens → Create API token**.

Файл `.env` не коммитится в git и не попадает никуда за пределы вашего компьютера.

## 2. Каждый раз, когда нужно создать проект

```powershell
cd e-sklep-b2b\agent
python run_form.py
```

- В терминале появится строка `Форма доступна на http://127.0.0.1:...`.
- Браузер откроется **сам** на нужном адресе — вручную открывать HTML-файл не нужно.
- Окно терминала с этой командой должно оставаться открытым, пока вы работаете с формой
  (это и есть сервер). Свёртывать можно, закрывать — нет.
- Если Windows Defender / антивирус спросит про доступ к сети для `python.exe` —
  разрешить (сервер работает только на `127.0.0.1`, за пределы компьютера не выходит).

## 3. В форме

1. Заполнить поля (название, заказчик, Jira Project Key, ERP, интеграции, особенности, даты).
2. Нажать **«Создать план (dry-run)»** — ничего не создаётся в Jira, только проверка.
3. Проверить отчёт. Если всё ок — появится кнопка **«Выполнить (создать в Jira)»**.
4. Нажать её только когда уверены — это реальное создание Epic/Issues/Sprints в Jira.

## Если что-то пошло не так

| Сообщение | Что делать |
|---|---|
| `python не является внутренней или внешней командой` | Python не установлен или не добавлен в PATH — см. шаг 0 |
| `ОШИБКА: не заданы переменные окружения...` в терминале | Шаг 1 не выполнен или `.env` заполнен неправильно |
| «Локальный сервер недоступен» в форме | Форма открыта не через `python run_form.py` — закрыть вкладку браузера, заново выполнить шаг 2 |
| Сервер «завис» и не отвечает | `Ctrl+C` в окне терминала, запустить `python run_form.py` снова |
| Терминал закрылся сам сразу после запуска | Прочитать сообщение об ошибке до закрытия — запустить из уже открытого PowerShell, а не двойным кликом по файлу, чтобы окно не закрывалось само |

**Главное правило:** форма всегда открывается только командой `python run_form.py` из папки
`e-sklep-b2b\agent` — никогда напрямую двойным кликом по HTML-файлу.
