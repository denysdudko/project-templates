## v2-предложение: OAuth-коннектор для Jira вместо env vars

Дата: 2026-07-09
Контекст: Stage 8 (Jira export) сейчас рассчитан на env vars
(JIRA_BASE_URL, JIRA_EMAIL, JIRA_API_TOKEN, JIRA_PROJECT_KEY)
в облачном окружении Claude Code.

Проблема: у облачных окружений нет отдельного secrets store —
любой соредактор окружения видит значения env vars, включая
JIRA_API_TOKEN. Приемлемо, пока окружение не расшарено.

Предложение: при появлении соавторов/коллег с доступом к
окружению — перейти на MCP-коннектор Atlassian Rovo (OAuth),
токен хранится на стороне Atlassian/Anthropic, а не в конфиге.

Стоимость перехода: Stage 8 придётся переписать под tool calls
коннектора (getJiraIssue, editJiraIssue и т.п.) вместо прямого
чтения переменных окружения — это меняет реализацию, не структуру
плана.

Статус: отложено. Не применять, пока окружение приватное.
